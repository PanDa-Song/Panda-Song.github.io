YelpCamp

Version 1

  1. Root/Home Page
  2. Campground Page lists all compgrounds, contains:
    • Name
    • Image
  3. Layout and Basic Styling
    • header and footer in partials
  4. Add ‘New Campground’ Page
    • ‘new campground’ POST route
    • add in body-parser (npm i body-parser)
    • setup route to show form and add form
  5. Style Navbar and Form with BootStrap 3

Version 2

  1. Add MongoDB and mongoose to the app

  2. Setup campground model and use it in the routes

     var campgroundSchema = new mongoose.Schema({
     name: String,
     image: String,
     description: String
 });

 var Campground = mongoose.model("Campground", campgroundSchema);
  3. Update the route to RESTful pattern
  4. Add ‘Show’ Page to show more info about one campground

Version 3

  1. Introduce the ‘module import’ method to clean the code of ‘app.js’
  2. Use ‘seedDB’ module to reset the DB for convinient test
  3. Add initial comments in ‘seedDB’ module and change ‘show’ page to show it.

Version 4

  1. Add RESTful Routes for ‘comments’ function including ‘new’ and ‘create’ and also comments page
  2. In views directory, make ‘campground’ directory to store the campground pages and make ‘comments’ directory to store comments pages.

Version 5

  1. Style the show page (grid model)
  2. Create a public directory to store the stylesheets

Version 6

  1. Using tools (node package)
    • passport
    • passport-local
    • passport-local-mongoose
  2. Add login, register and logout function

Version 7

  1. Use express router to reorgnized all routes. Every routes become a sigle file and be used in ‘app.js’

Version 8

  1. Associate users and comments
  2. Save author’s name to a comment automatically

Version 9

  1. Prevent an unauthenticated user from creating a campground
  2. Save username + id to newly created campground

Version 10

  1. Add edit/update function and destroy function for both campgrounds and comments.
  2. Add authorization function for edit and delete operation
  3. Refactor the middlewares to one file

Version 11

  1. Add the meaning for error message using the package ‘connect-flash’ and bootstrap components ‘alert’.
  2. Refactor Landing Page(HOME PAGE)
  3. Add dynamic price feature
  4. Deploy to heroku with MongoAtlas
