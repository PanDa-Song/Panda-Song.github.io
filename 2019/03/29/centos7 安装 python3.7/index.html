centos7 安装 python3.7
发表于2019-03-29|更新于2020-01-01|入门
|阅读量:

1. 下载

官网 https://www.python.org/downloads/ ，选择source code版本下载到linux本地

2. 解压安装

先建立安装目录
#mkdir -p /usr/local/python3 (用root身份)

将刚刚下载的python的tar文件解压到此
#tar -zxvf Python-3.7.3.tgz

3. 编译

进入解压后的目录
#cd Python-3.7.3
编译安装前安装一个包，否则pip的安装会报错
#yum install libffi-devel -y
编译安装
#.configure --prefix=/usr/local/python3
#make
#make install

4. 建立python3软链，并将python3加入PATH

建立python3软链
#ln -s /usr/local/python3/bin/python3 /usr/bin/python3
用vim修改PATH文件

# vim ~/.bash_profile
(以下是文件内容)
# .bash_profile
# Get the aliases and functions
if [ -f ~/.bashrc ]; then
. ~/.bashrc
fi
# User specific environment and startup programs
PATH=$PATH:$HOME/bin:/usr/local/python3/bin
export PATH

执行如下命令使其生效
#source ~/.bash_profile

5. 安装完成

检查安装情况

#python3 -V
python-3.7.3
#pip3 -V
pip-19.0.3  from ...

但是，由于linux系统安装是自带有python2，所以即使此时python3已经安装完毕，在终端输入python仍然会进入的是系统自带的python2，只有输入python3才能进入python3编译器，因此为了之后使用的方便，可以适当的建立软链接来改善。

#ln -s /usr/local/python3/bin/python3 /usr/bin/python
# ln -s /usr/local/python3/bin/pip3 /usr/bin/pip

改完之后，由于linux下yum系统借助的是自带的python，所以改成3.7后会导致yum无法正常使用。需要进行修改如下：

#vim /usr/bin/yum
修改第一行Python为Python2.7
