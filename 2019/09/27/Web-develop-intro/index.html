Intro

HTML

  • HyperText Markup Language
  • Define the structure of a webpage
  • ‘noun’ of webpage

    CSS

  • Cascading Style Sheets
  • Define the style of HTML
  • ‘adj’ of webpage

    JavaScript

  • Adds logic and interactivity to a page
  • ‘verbs’ of webpage

HTML

intro

General Rule:


<tagname> Some Content </tagname>

But Every HTML documents will start with this boilerplate:

<!DOCTYPE html>
<html>
<head>
    <title></title>
</head>
<body>

</body>
</html>

In sublime, type ‘html’ following a Tab this structure will appear.

Basic tags

Comment: <!-- xxxx -->(cltr+/)


Line break: <p>


Bold: <b> or <strong>


Italic: <i> or <em>(emphasis)

Html List

  • Order list(with number)
    <ol>
  <li>apple</li>
  <li>orange</li>
  <li>peer</li>
</ol>
  • Unorder list(with bold points)
    <ul>
  <li>apple</li>
  <li>orange</li>
  <li>peer</li>
</ul>

Forms

  • The <form>tag


    container of inputs

    <form action="/my-form-submitting-page" method="post">
  <!-- All inputs will go in here -->
</form>

  • action - the URL to send from data to

  • method - the type of HTTP request
    will illustrate in BackEnd

  • The <input> tag


    create interactive controls. The “type” attribute determines the type of input

    <input type="text">
<input type="date">
<input type="color">
<input type="file">
<input type="checkbox">

A simple form example:

<h1>Sign in</h1>
    <form action="/sign-in-url" method="post">
        <label>
            Username: 
            <input type="text" placeholder="Account">
        </label>
        <label>
            Password: 
            <input type="password" name="password">
        </label>
        <button>
            Log in
        </button>
    </form>

Labels

Showed above


Syntax:

<!-- first type (normal type) -->

<label>
    Username: 
    <input type="text" placeholder="Account">
</label>

<!-- other type of syntax using 'for' and 'id'-->

<label for="username">Username: </label>
<input id="username" type="text" placeholder="Account">


CSS

Intro

  • Cascading Style Sheets

    • the adj. for a web page
    • to modify the html

  • General Rule

    selctor {
    property: value;
    anotherProperty: value;
}

    Examples: 

    /*Make all h1 purple and 56px font*/
h1 {
    color: purple;
    font-size: 56px;
}
/*Give all image a 3px red border*/
img {
    border-color: red;
    border-width: 3px;
}

  • Where to put CSS

    • inline(not good: need to edit for every elements, we want to seperate CSS and html)
    • Style tag
      General rule:
      <style type="text/css">
    selector {
        property: value;
    }
</style>

    Both above are not definately seperate CSS and HTML, so using another way.

    • Write CSS in a seperate CSS file (using <link> tage):

      <link rel="stylesheet" type="text/css" href="app.css">(in html file)

  • Color in CSS

    1. Hexadecimal(16 number):


      # + String of 6 hexadecimal numbers(from 0 - F)


      first two digits –> how much red;


      next two digits –> how much green;


      last two digits –> how much blue;


      Generally using color picker

    2. RGB:


      3 channels: Red, Green, Blue. Each ranges from 0-255


      color: rgb(0, 255, 0);

    3. RGBA:


      just like RGB, but with an alpha(transparency) channel from 0.0 - 1.0(lower means sligter)


      color: rgba(11, 99, 150, 1);

  • Background and Border

    • Background

      • color:


        background: rgb(255, 0, 0);

      • image:

        background: url(image address)
background-repeat: no-repeat;
background-size: cover;

    • Border


      Three parts:

        1. width
  2. color
  3. style
      /*Syntax: color width style */
board: purple 5px solid;

  • CSS Selector

    • Element Selector: Select all instances of a given element
    • ID Selector: Select an element with a given ID. One ID can be only once in one page.
      <!--html file-->
<li id="special">
      /*CSS file*/
#special{
background: yellow;
}
    • Class Selector: Similar to ID, but change the word ID to class and ‘#’ to ‘.’

  • Chrome Inspector

  • More Advanced Selectors

    1. Element
    2. class
    3. name
    4. star
    5. descendant selector
    6. adjacent selector
    7. attribute selector
    8. nth of type

  • Specificity

    • inherit
    • specificity: one element inherit from different parents, specific one will win.
